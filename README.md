# X投稿管理（x-post-manager）

X（旧Twitter）のデータアーカイブから投稿一覧を取得し、Google スプレッドシートで管理するツールです。

- **スプレッドシート上にボタン**を配置し、ZIP または `tweets.js` をアップロードして取り込み
- **重複時は変更がある列だけ上書き**（いいね数・リツイート数などが更新された分だけ反映）
- API 不要（X の「データのアーカイブをダウンロード」で取得したファイルのみで利用可能）

---

## 使い方（スプレッドシート）

1. **アーカイブの取得**  
   X にログイン → 設定とプライバシー → アカウント → 「データのアーカイブをダウンロード」で ZIP をリクエスト（準備に24時間程度かかることがあります）。

2. **スプレッドシートに GAS を紐づける**
   - 新しい Google スプレッドシートを作成する
   - 拡張機能 → Apps Script を開く
   - `gas/Code.gs` の内容をそのままコピーして、デフォルトの `Code.gs` を置き換える
   - ファイル → 新規 → HTML ファイル → 名前を `sidebar` にし、`gas/sidebar.html` の内容を貼り付けて保存
   - 保存後、スプレッドシートを再読み込みする

3. **取り込み**
   - スプレッドシートのメニューに「X投稿管理」→「アーカイブをインポート」が表示される
   - クリックでサイドバーが開くので、ZIP または `tweets.js` を選択して「取り込み実行」を押す
   - 同じシート内に「投稿一覧」シートが自動作成され、1行1投稿で書き出される（既に同じ ID の行がある場合は、変更がある列だけ上書き）

## clasp でのデプロイ

コードをローカルで編集し、clasp で Apps Script に反映する手順です。

### 初回のみ（紐づけ）

1. **依存関係のインストール**
   ```bash
   npm install
   ```

2. **Google アカウントでログイン**
   ```bash
   npx clasp login
   ```
   ブラウザが開くので、使う Google アカウントで許可してください。

3. **スプレッドシートとスクリプトの準備**
   - 新しい Google スプレッドシートを作成する
   - メニュー **拡張機能** → **Apps Script** を開く
   - 左の **プロジェクトの設定**（歯車アイコン）を開き、**スクリプト ID** をコピーする

4. **このリポジトリを Apps Script と紐づける**
   ```bash
   npx clasp clone <スクリプトID> --rootDir ./gas
   ```
   ※ `<スクリプトID>` をコピーした ID に置き換えてください。  
   これでプロジェクト直下に `.clasp.json` が作成されます（Git には含めません）。

5. **初回プッシュ**
   ```bash
   npm run gas:push
   ```
   `gas/` 内の `Code.gs`・`sidebar.html`・`appsscript.json` が Apps Script にアップロードされます。

6. **スプレッドシートを再読み込み**
   ブラウザでスプレッドシートを更新すると、メニューに「X投稿管理」→「アーカイブをインポート」が表示されます。

### 2回目以降（コードを更新したとき）

- **プッシュ（開発用・「Head」に反映）**
  ```bash
  npm run gas:push
  ```
  保存後、スプレッドシートを再読み込みすれば変更が反映されます。

- **デプロイ（バージョン付きで本番用として出す場合）**
  ```bash
  npm run gas:deploy
  ```
  通常の利用では `gas:push` だけで十分です。

## 運用方法（日々の使い方）

| やりたいこと | 手順 |
|--------------|------|
| **新しいアーカイブを取り込む** | スプレッドシートで **X投稿管理** → **アーカイブをインポート** を開き、ZIP または `tweets.js` を選んで **取り込み実行**。同じ ID の投稿は変更があった列だけ上書きされます。 |
| **過去に取り込んだアーカイブを再取り込みする** | 上と同じ。同じ投稿は「いいね数・リツイート数」など変更分だけ更新されます。 |
| **GAS のコードを直した・機能を足した** | ローカルで `gas/` を編集 → `npm run gas:push` → スプレッドシートを再読み込み。 |
| **別のスプレッドシートでも使いたい** | そのシートで **拡張機能** → **Apps Script** を開き、新しいスクリプトを作成。**スクリプト ID** をコピーして、別フォルダで `npx clasp clone <スクリプトID> --rootDir ./gas` するか、または手動で `Code.gs` と `sidebar.html` をコピーして貼り付け。 |

**注意**

- 取り込みは「そのスプレッドシートに紐づいた Apps Script」で動きます。clasp で push したスクリプトと、使っているスプレッドシートが同じスクリプト ID のものであることを確認してください。
- アーカイブは X の **設定とプライバシー** → **アカウント** → **データのアーカイブをダウンロード** で取得した ZIP、またはその中の `tweets.js` を指定してください。

## 開発（Node / CLI）

```bash
cd x-post-manager
npm install
```

- **CLI でパースのみ（CSV 出力）**  
  `npx ts-node src/cli.ts parse ./path/to/tweets.js`  
  または ZIP のパスを指定してもよい。

## スプレッドシートの列

| 列 | 説明 |
|----|------|
| id | 投稿ID（ツイートID） |
| text | 本文 |
| created_at | 投稿日時 |
| url | 投稿の permalink |
| source | クライアント名 |
| is_retweet / is_reply | リツイート・リプライか |
| like_count / retweet_count | いいね数・リツイート数 |
| urls / hashtags / mentions / media | 本文内URL・ハッシュタグ・メンション・メディアURL（カンマ区切り） |
